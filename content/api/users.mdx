# Users API

User management endpoints are part of the **Auth service** (`http://localhost:8081`), not the Dashboard API.

All endpoints require a valid Bearer token. Write operations require specific admin grants.

---

## Get All Users

```http
GET /api/v2/user/
Authorization: Bearer <access_token>
```

### Response `200 OK`

```json
[
  {
    "id": "507f1f77bcf86cd799439011",
    "email": "john@example.com",
    "emailVerified": true,
    "emailVerifiedAt": "2024-01-10T09:00:00",
    "locked": false,
    "failedLoginAttempts": 0,
    "profileImageUrl": null,
    "roles": [{ "id": "...", "name": "admin", "grants": [] }],
    "createdAt": "2024-01-01T00:00:00Z",
    "deletedAt": null
  }
]
```

---

## Search Users

Case-insensitive partial match on email.

```http
GET /api/v2/user/search?q=john
Authorization: Bearer <access_token>
```

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `q` | string | Yes | Search query matched against email |

---

## Get User by ID

```http
GET /api/v2/user/{id}
Authorization: Bearer <access_token>
```

Returns `404` if not found.

---

## Update User

**Required Grant**: `dashboard-oauth-user-update`

```http
PUT /api/v2/user/{id}
Authorization: Bearer <access_token>
Content-Type: application/json
```

```json
{
  "email": "newemail@example.com"
}
```

Returns the updated `UserAdminRead`. `409` if the email is already in use.

---

## Delete User

Soft-deletes the user account.

**Required Grant**: `dashboard-oauth-user-delete`

```http
DELETE /api/v2/user/{id}
Authorization: Bearer <access_token>
```

Returns `204 No Content`.

---

## Block User

Locks the account, preventing login.

**Required Grant**: `dashboard-oauth-user-block`

```http
POST /api/v2/user/{id}/block
Authorization: Bearer <access_token>
```

Returns `204 No Content`. `409` if already blocked.

---

## Unblock User

Unlocks the account and resets failed login attempt counter.

**Required Grant**: `dashboard-oauth-user-block`

```http
POST /api/v2/user/{id}/unblock
Authorization: Bearer <access_token>
```

Returns `204 No Content`. `409` if not blocked.

---

## Resend Verification Email

Queues a new verification email (sent on the next scheduler run).

**Required Grant**: `dashboard-oauth-user-resend-verification`

```http
POST /api/v2/user/{id}/resend-verification
Authorization: Bearer <access_token>
```

Returns `204 No Content`. `409` if email is already verified.

---

## Trigger Password Reset

Queues a password reset email for the user (sent on the next scheduler run).

**Required Grant**: `dashboard-oauth-user-reset-password`

```http
POST /api/v2/user/{id}/reset-password
Authorization: Bearer <access_token>
```

Returns `204 No Content`.
