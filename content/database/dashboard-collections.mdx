# Dashboard Collections

Collections used by the main Dashboard API service.

## customers

Stores customer information.

```typescript
{
  _id: string,
  name: string,
  email: string,      // unique index
  imageId?: string,   // reference to uploaded image (optional)
  audit: {
    createdAt: string,
    updatedAt: string,
    deletedAt: string | null
  }
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `_id` | ObjectId | Yes | Primary key |
| `name` | string | Yes | Customer full name |
| `email` | string | Yes | Unique email address |
| `imageId` | ObjectId | No | ObjectId of the uploaded profile image |
| `audit` | Audit | Yes | Timestamps |

> **Note**: `imageId` stores a reference to an object in cloud storage (R2). The resolved public URL is returned as `image_url` in API responses by the mapper layer.

### Example Document

```json
{
  "_id": { "$oid": "507f1f77bcf86cd799439011" },
  "name": "John Doe",
  "email": "john@example.com",
  "imageId": { "$oid": "507f1f77bcf86cd799439099" },
  "audit": {
    "createdAt": { "$date": "2024-01-15T10:30:00Z" },
    "updatedAt": { "$date": "2024-01-15T10:30:00Z" },
    "deletedAt": null
  }
}
```

## invoices

Stores invoice records with references to customers.

```typescript
{
  _id: string,
  customer: { $ref: string; $id: string },  // DBRef to customers
  amount: number,
  date: string,     // ISO 8601 date string e.g. "2024-01-15"
  status: 'pending' | 'paid',
  audit: {
    createdAt: string,
    updatedAt: string,
    deletedAt: string | null
  }
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `_id` | ObjectId | Yes | Primary key |
| `customer` | DBRef | Yes | Reference to customer |
| `amount` | Double | Yes | Invoice amount |
| `date` | LocalDate | Yes | Invoice date |
| `status` | string | Yes | `pending` or `paid` |
| `audit` | Audit | Yes | Timestamps |

### Example Document

```json
{
  "_id": { "$oid": "507f1f77bcf86cd799439012" },
  "customer": {
    "$ref": "customers",
    "$id": { "$oid": "507f1f77bcf86cd799439011" }
  },
  "amount": 1500.00,
  "date": "2024-01-15",
  "status": "pending",
  "audit": {
    "createdAt": { "$date": "2024-01-15T10:30:00Z" },
    "updatedAt": { "$date": "2024-01-15T10:30:00Z" },
    "deletedAt": null
  }
}
```

### Status Values

| Status | Description |
|--------|-------------|
| `pending` | Invoice awaiting payment |
| `paid` | Invoice has been paid |

## revenues

Stores monthly revenue aggregations for dashboard charts. Has a compound unique index on `(month, year)`.

```typescript
{
  _id: string,
  month: 'JANUARY' | 'FEBRUARY' | 'MARCH' | 'APRIL' | 'MAY' | 'JUNE'
       | 'JULY' | 'AUGUST' | 'SEPTEMBER' | 'OCTOBER' | 'NOVEMBER' | 'DECEMBER',
  year: number,   // 2000â€“9999
  revenue: number,
  audit: {
    createdAt: string,
    updatedAt: string,
    deletedAt: string | null
  }
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `_id` | ObjectId | Yes | Primary key |
| `month` | string (enum) | Yes | Month name e.g. `"JANUARY"` |
| `year` | integer | Yes | 4-digit year (2000â€“9999) |
| `revenue` | Decimal | Yes | Total revenue for that month |
| `audit` | Audit | Yes | Timestamps |

### Example Document

```json
{
  "_id": { "$oid": "507f1f77bcf86cd799439013" },
  "month": "JANUARY",
  "year": 2024,
  "revenue": 15000.00,
  "audit": {
    "createdAt": { "$date": "2024-01-31T23:59:59Z" },
    "updatedAt": { "$date": "2024-01-31T23:59:59Z" },
    "deletedAt": null
  }
}
```

## invoices_search

A denormalised search index kept in sync with the `invoices` collection. Used to power the paginated invoice search with MongoDB text indexes across invoice and customer fields.

```typescript
{
  _id: string,
  invoiceId: string,        // reference to invoices._id (indexed)
  customerId: string,       // reference to customers._id (indexed)
  amount: number,
  date: string,             // ISO 8601 date string
  status: string,           // text-indexed, weight 2
  customerName: string,     // text-indexed, weight 3
  customerEmail: string,    // text-indexed, weight 2
  customerImageUrl: string,
  deletedAt: string | null,
  lastSyncedAt: string | null
}
```

| Index | Fields | Type |
|-------|--------|------|
| `customer_idx` | `customerId` | Compound |
| `deletedAt_idx` | `deletedAt` | Compound |
| text index | `status`, `customerName`, `customerEmail` | Text |
