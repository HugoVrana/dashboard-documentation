# OAuth Collections

Collections used by the OAuth authentication service.

## users

Stores user authentication information.

```typescript
{
  _id: string,
  email: string,                // unique index
  password: string,             // BCrypt hashed
  emailVerified: boolean,
  emailVerifiedAt: string | null,   // ISO 8601 timestamp
  emailVerificationToken: { token: string; expiresAt: string } | null,
  passwordResetToken: { token: string; expiresAt: string } | null,
  profileImageId?: string,      // optional
  failedLoginAttempts: number,  // default 0
  locked: boolean,
  twoFactorConfig: { secret: string; verified: boolean } | null,
  roles: { $ref: string; $id: string }[],  // DBRefs to roles
  audit: {
    createdAt: string,
    updatedAt: string,
    deletedAt: string | null
  }
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `_id` | ObjectId | Yes | Primary key |
| `email` | string | Yes | Unique email address |
| `password` | string | Yes | BCrypt hashed password |
| `emailVerified` | Boolean | Yes | Verification status |
| `emailVerifiedAt` | DateTime | No | Verification timestamp |
| `emailVerificationToken` | Token | No | Pending verification token |
| `passwordResetToken` | Token | No | Pending password reset token |
| `profileImageId` | ObjectId | No | Reference to uploaded profile image |
| `failedLoginAttempts` | integer | Yes | Failed login count (default 0) |
| `locked` | boolean | Yes | Whether account is locked |
| `twoFactorConfig` | TotpConfig | No | TOTP configuration if 2FA is enabled |
| `roles` | DBRef[] | Yes | Assigned roles |
| `audit` | Audit | Yes | Timestamps |

### Example Document

```json
{
  "_id": { "$oid": "507f1f77bcf86cd799439015" },
  "email": "user@example.com",
  "password": "$2a$10$N9qo8uLOickgx2ZMRZoMyeIjZRGdjGj/n6F...",
  "emailVerified": true,
  "emailVerifiedAt": { "$date": "2024-01-15T12:00:00Z" },
  "emailVerificationToken": null,
  "passwordResetToken": null,
  "roles": [
    {
      "$ref": "roles",
      "$id": { "$oid": "507f1f77bcf86cd799439016" }
    }
  ],
  "audit": {
    "createdAt": { "$date": "2024-01-15T10:00:00Z" },
    "updatedAt": { "$date": "2024-01-15T12:00:00Z" },
    "deletedAt": null
  }
}
```

## roles

Stores role definitions with grant references.

```typescript
{
  _id: string,
  name: string,           // unique index
  grants: { $ref: string; $id: string }[],  // DBRefs to grants
  audit: {
    createdAt: string,
    updatedAt: string,
    deletedAt: string | null
  }
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `_id` | ObjectId | Yes | Primary key |
| `name` | string | Yes | Unique role name |
| `grants` | DBRef[] | Yes | Assigned permissions |
| `audit` | Audit | Yes | Timestamps |

### Example Document

```json
{
  "_id": { "$oid": "507f1f77bcf86cd799439016" },
  "name": "admin",
  "grants": [
    { "$ref": "grants", "$id": { "$oid": "507f1f77bcf86cd799439017" } },
    { "$ref": "grants", "$id": { "$oid": "507f1f77bcf86cd799439018" } }
  ],
  "audit": {
    "createdAt": { "$date": "2024-01-01T00:00:00Z" },
    "updatedAt": { "$date": "2024-01-10T15:30:00Z" },
    "deletedAt": null
  }
}
```

## grants

Stores individual permission definitions.

```typescript
{
  _id: string,
  name: string,           // unique index
  description: string,
  audit: {
    createdAt: string,
    updatedAt: string,
    deletedAt: string | null
  }
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `_id` | ObjectId | Yes | Primary key |
| `name` | string | Yes | Unique permission name |
| `description` | string | No | Human-readable description |
| `audit` | Audit | Yes | Timestamps |

### Example Document

```json
{
  "_id": { "$oid": "507f1f77bcf86cd799439017" },
  "name": "dashboard-invoices-read",
  "description": "Permission to read invoice data",
  "audit": {
    "createdAt": { "$date": "2024-01-01T00:00:00Z" },
    "updatedAt": { "$date": "2024-01-01T00:00:00Z" },
    "deletedAt": null
  }
}
```

### Standard Grants

| Grant Name | Description |
|------------|-------------|
| `dashboard-customers-read` | Read customer data |
| `dashboard-invoices-read` | Read invoice data |
| `dashboard-invoices-create` | Create invoices |
| `dashboard-invoices-update` | Update invoices |
| `dashboard-invoices-delete` | Delete invoices |
| `dashboard-revenue-read` | Read revenue data |
| `dashboard-users-read` | Read user data |
| `dashboard-users-create` | Create/search users |

## refresh_tokens

Stores active refresh tokens for token rotation.

```typescript
{
  _id: string,
  token: string,
  userId: string,
  expiresAt: string,   // ISO 8601 timestamp
  createdAt: string
}
```

---

## email_send_attempts

Tracks email sending history for verification and password reset.

```typescript
{
  _id: string,
  userId: string,
  emailType: 'VERIFICATION' | 'PASSWORD_RESET',
  status: 'PENDING' | 'SENT' | 'FAILED',
  sentAt: string,     // ISO 8601 timestamp
  error: string | null
}
```

### Email Types

| Type | Description |
|------|-------------|
| `VERIFICATION` | Email verification |
| `PASSWORD_RESET` | Password reset |

### Email Status

| Status | Description |
|--------|-------------|
| `PENDING` | Queued for sending |
| `SENT` | Successfully sent |
| `FAILED` | Send failed |

## oauth_clients

Registered OAuth 2.0 clients allowed to initiate the authorization flow.

```typescript
{
  _id: string,
  clientId: string,       // unique
  clientSecretHash: string,
  allowedOrigins: string[],
  redirectUris: string[],
  createdAt: string       // ISO 8601 timestamp
}
```

---

## authorization_requests

Short-lived records representing in-progress authorization requests (created by `GET /oauth2/authorize`, consumed after login).

```typescript
{
  _id: string,
  clientId: string,
  redirectUri: string,
  codeChallenge: string,
  codeChallengeMethod: string,
  scope: string,
  state: string,
  nonce: string,
  createdAt: string     // ISO 8601 timestamp
}
```

---

## authorization_codes

Single-use authorization codes issued after successful login, exchanged for tokens at `/oauth2/token`.

```typescript
{
  _id: string,
  code: string,           // unique
  clientId: string,
  userId: string,
  redirectUri: string,
  codeChallenge: string,
  scope: string,
  state: string,
  nonce: string,
  expiresAt: string,      // ISO 8601 timestamp
  usedAt: string | null
}
```

---

## oauth_tokens

Issued access tokens.

```typescript
{
  _id: string,
  token: string,          // unique
  userId: string,
  clientId: string,
  scope: string,
  expiresAt: string,      // ISO 8601 timestamp
  revokedAt: string | null
}
```

---

## mfa_tokens

Short-lived tokens used during the MFA step of the authorization flow. Created when login succeeds but the account has TOTP enabled; consumed when the TOTP code is verified.

```typescript
{
  _id: string,
  token: string,              // unique
  authorizationCodeId: string,
  expiresAt: string           // ISO 8601 timestamp
}
```

---

## server_keys

RSA key pairs used to sign JWTs. The active key is rotated on a schedule.

```typescript
{
  _id: string,
  kid: string,        // key ID included in JWT header
  publicKey: string,  // PEM encoded
  privateKey: string, // PEM encoded, encrypted at rest
  createdAt: string,  // ISO 8601 timestamp
  expiresAt: string
}
```
