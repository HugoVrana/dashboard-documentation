# Frontend Integration

The frontend uses **NextAuth.js** for session management combined with a custom OAuth2 PKCE flow to authenticate against the auth service.

## Login Flow

Authentication is handled through Next.js server actions in `app/auth/actions/loginActions.ts`. The flow follows OAuth2 Authorization Code + PKCE:

```
Frontend                           Auth Service
   │                                    │
   │── initiatePkceAction() ──────────► │ GET /api/v2/oauth2/authorize
   │                                    │   validates client, creates request
   │◄── request_id ─────────────────── │ 302 → login page?request_id=...
   │                                    │
   │── loginAction(email, password) ──► │ POST /api/v2/oauth2/authorize
   │                                    │   validates credentials
   │                              ┌─────┤
   │                        MFA?  │ No  │ 302 → redirect?code=...
   │                              │ Yes │ 200 { mfa_required, mfa_token }
   │                              └─────┤
   │── completeMfaLoginAction(totp) ──► │ POST /api/v2/oauth2/authorize/mfa
   │                                    │ 302 → redirect?code=...
   │                                    │
   │── completePkceCallbackAction() ──► │ POST /api/v2/oauth2/token
   │◄── { access_token, refresh_token } │
   │── signIn() → NextAuth session ───► session stored
```

PKCE state (code verifier) and MFA tokens are stored in **encrypted httpOnly cookies** (`pkce_state`, `mfa_pending`) between steps using AES-256-GCM.

## NextAuth Configuration

NextAuth manages the session after the OAuth2 flow completes. The JWT callback stores the access token, refresh token, roles, and grants.

```typescript
// app/auth/auth.ts (approximate structure)
export const { handlers, signIn, signOut, auth } = NextAuth({
  callbacks: {
    async jwt({ token, user }) {
      if (user) {
        token.accessToken = user.accessToken;
        token.refreshToken = user.refreshToken;
        token.expiresAt = Date.now() + user.expiresIn * 1000;
        token.role = user.role;
        token.grants = user.grants;
        token.url = user.url;
      }
      return token;
    },

    async session({ session, token }) {
      session.accessToken = token.accessToken;
      session.refreshToken = token.refreshToken;
      session.expiresAt = token.expiresAt;
      session.url = token.url;
      session.user.id = token.id;
      session.user.role = token.role;
      session.user.grants = token.grants;
      return session;
    }
  },

  pages: {
    signIn: '/auth/login',
    error: '/auth/unauthorized'
  }
});
```

## Session Shape

```typescript
// app/auth/models/next-auth.d.ts
interface Session {
  accessToken: string;
  refreshToken: string;
  expiresAt: number;        // Unix ms timestamp
  url: string;              // Auth service base URL
  user: {
    id: string;
    role: Role[];
    grants: string[];       // flattened permission names
    image?: string | null;
  } & DefaultSession['user'];
}
```

## Using the Session

### Server Components

```typescript
import { auth } from '@/auth';
import { redirect } from 'next/navigation';

export default async function ProtectedPage() {
  const session = await auth();

  if (!session) {
    redirect('/auth/login');
  }

  const invoices = await getAllInvoices(session.accessToken);
  return <InvoiceList invoices={invoices} />;
}
```

### Client Components

```typescript
'use client';
import { useSession } from 'next-auth/react';

export function UserMenu() {
  const { data: session } = useSession();
  if (!session) return null;
  return <span>{session.user.email}</span>;
}
```

## Permission Checking

Grants are stored flat in the session — no extra API call needed:

```typescript
import { auth } from '@/auth';

export default async function InvoicesPage() {
  const session = await auth();
  const canCreate = session?.user.grants.includes('dashboard-invoices-create');
  const canDelete = session?.user.grants.includes('dashboard-invoices-delete');

  return (
    <div>
      {canCreate && <CreateInvoiceButton />}
      <InvoiceTable canDelete={canDelete} />
    </div>
  );
}
```

## Token Refresh

Handled automatically in the NextAuth JWT callback: `expiresAt` is compared against the current time, and `POST /api/v2/oauth2/token` with `grant_type=refresh_token` is called when the access token has expired. The old refresh token is invalidated (rotation).

## TOTP Setup

After registration, if `requiresTwoFactorEnrollment: true`, the frontend calls:

```typescript
// app/auth/actions/totpActions.ts
setupTotp(accessToken: string): Promise<TotpSetupResponse>
// Returns { qrCodeDataUri, secret }

verifyTotp(code: string, accessToken: string): Promise<boolean>
```

The QR code is shown in `totpSetupStep.tsx`. After scanning and entering the 6-digit code, 2FA is activated on the account.
