# Authentication Overview

The Dashboard uses a separate OAuth service for authentication, implementing JWT-based authentication with refresh token rotation.

## Authentication Flow

```
┌──────────┐     1. Login Request      ┌──────────────┐
│          │ ────────────────────────► │              │
│  Client  │                           │ OAuth Service│
│          │ ◄──────────────────────── │              │
└──────────┘  2. JWT + Refresh Token   └──────────────┘
     │
     │ 3. API Request with JWT
     ▼
┌──────────────┐
│              │
│ Dashboard API│
│              │
└──────────────┘
```

## Key Features

- **JWT Access Tokens** - Short-lived tokens for API access
- **Refresh Token Rotation** - Secure token refresh with rotation
- **Role-Based Access** - Users assigned roles with specific grants
- **Email Verification** - Verify user email addresses
- **Password Reset** - Secure password reset flow
