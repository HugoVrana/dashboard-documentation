# Authentication Overview

The Dashboard uses a dedicated OAuth service for authentication, implementing the OAuth 2.0 Authorization Code flow with PKCE and OpenID Connect.

## Authentication Flow

```
┌──────────┐  1. GET /oauth2/authorize   ┌──────────────┐
│          │ ──────────────────────────► │              │
│  Client  │  2. 302 → login page        │ OAuth Service│
│          │ ◄────────────────────────── │  (port 8081) │
│          │                             │              │
│          │  3. POST /oauth2/authorize  │              │
│          │ ──────────────────────────► │              │
│          │  4. 302 → redirect?code=... │              │
│          │ ◄────────────────────────── │              │
│          │                             │              │
│          │  5. POST /oauth2/token      │              │
│          │ ──────────────────────────► │              │
│          │  6. access + refresh tokens │              │
│          │ ◄────────────────────────── └──────────────┘
│          │
│          │  7. API request + Bearer token
│          │ ──────────────────────────► ┌──────────────┐
│          │                             │ Dashboard API│
│          │ ◄────────────────────────── │  (port 8080) │
└──────────┘                             └──────────────┘
```

If MFA is enabled on the account, step 3 returns an `mfa_token` and the client must submit a TOTP code before receiving the authorization code.

## Key Features

- **OAuth 2.0 + PKCE** - Authorization Code flow (RFC 6749 + RFC 7636)
- **OpenID Connect** - UserInfo endpoint and ID tokens
- **TOTP / 2FA** - Optional per-account two-factor authentication
- **Refresh Token Rotation** - Old refresh tokens are invalidated on use
- **Email Verification** - Account activation via email
- **Password Reset** - Self-service password reset via email
- **Role-Based Access** - Users assigned roles with specific grants
- **Account Locking** - Automatic locking after failed login attempts
