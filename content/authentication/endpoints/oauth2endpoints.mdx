# OAuth2 Endpoints

Base URL: `/api/auth`

## Register

Creates a new user account.

```http
POST /api/auth/register
Content-Type: application/json
```

### Request

```json
{
  "email": "user@example.com",
  "password": "securePassword123",
  "roleId": "507f1f77bcf86cd799439011"
}
```

### Response `201 Created`

```json
{
  "id": "507f1f77bcf86cd799439012",
  "email": "user@example.com",
  "roleReads": [
    {
      "id": "507f1f77bcf86cd799439011",
      "name": "user",
      "grants": [
        {
          "id": "507f1f77bcf86cd799439013",
          "name": "dashboard-invoices-read",
          "description": "Read invoice data"
        }
      ]
    }
  ]
}
```

### Response `400 BAD REQUEST`

Invalid request data or validation failed.

### Response `404 NOT FOUND`

Role ID does not exist.

### Response `409 CONFLICT`

A user with this email already exists.

---

## Login

Authenticates a user and returns tokens.

```http
POST /api/auth/login
Content-Type: application/json
```

### Request

```json
{
  "email": "user@example.com",
  "password": "securePassword123"
}
```

### Response `200 OK`

```json
{
  "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refreshToken": "dGhpcyBpcyBhIHJlZnJlc2ggdG9rZW4...",
  "user": {
    "id": "507f1f77bcf86cd799439012",
    "email": "user@example.com",
    "roleReads": [...]
  }
}
```

### Response `401 UNAUTHORIZED`

Invalid email or password.

---

## Refresh Token

Exchanges a refresh token for new tokens.

```http
POST /api/auth/refresh
Content-Type: application/json
```

### Request

```json
{
  "refreshToken": "dGhpcyBpcyBhIHJlZnJlc2ggdG9rZW4..."
}
```

### Response `200 OK`

```json
{
  "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refreshToken": "bmV3IHJlZnJlc2ggdG9rZW4..."
}
```

> **Note**: The old refresh token is invalidated (rotation).

### Response `401 UNAUTHORIZED`

Invalid or expired refresh token.

---

## Logout

Invalidates the user's refresh tokens.

```http
POST /api/auth/logout
Authorization: Bearer <access_token>
```

### Response `200 OK`

No body.

### Response `401 UNAUTHORIZED`

Missing or invalid access token.

---

## Get Current User

Returns the authenticated user's information.

```http
GET /api/auth/me
Authorization: Bearer <access_token>
```

### Response `200 OK`

```json
{
  "id": "507f1f77bcf86cd799439012",
  "email": "user@example.com",
  "roleReads": [...]
}
```

### Response `401 UNAUTHORIZED`

Missing or invalid access token.

---

## Verify Email

Verifies the user's email address.

```http
POST /api/auth/verify-email?token=verification_token_here
```

### Response `200 OK`

No body.

### Response `400 BAD REQUEST`

Invalid or expired verification token.

---

## Forgot Password

Initiates the password reset flow.

```http
POST /api/auth/forgot-password
Content-Type: application/json
```

### Request

```json
{
  "email": "user@example.com"
}
```

### Response `200 OK`

No body. Email sent if user exists.

---

## Reset Password

Resets the user's password.

```http
POST /api/auth/reset-password
Content-Type: application/json
```

### Request

```json
{
  "token": "password_reset_token",
  "newPassword": "newSecurePassword456"
}
```

### Response `200 OK`

No body.

### Response `400 BAD REQUEST`

Invalid or expired reset token.

---

## Add Role to User

Assigns an additional role to a user.

```http
POST /api/auth/user/role
Content-Type: application/json
```

### Request

```json
{
  "userId": "507f1f77bcf86cd799439012",
  "roleId": "507f1f77bcf86cd799439011"
}
```

### Response `200 OK`

```json
{
  "user": {
    "id": "507f1f77bcf86cd799439012",
    "email": "user@example.com",
    "roleReads": [...]
  }
}
```

### Response `400 BAD REQUEST`

Invalid user ID or role ID format.

### Response `404 NOT FOUND`

User or role not found.

### Response `409 CONFLICT`

User already has this role assigned.

---

## Error Responses

All endpoints may return these common error codes:

| Code | Exception | Description |
|------|-----------|-------------|
| `400` | `InvalidRequestException` | Invalid request data or validation failed |
| `401` | `ResponseStatusException` | Invalid credentials or missing/expired token |
| `404` | `ResourceNotFoundException` | Requested resource not found |
| `409` | `ConflictException` | Resource conflict (e.g., duplicate email, role already assigned) |
| `500` | `Exception` | Internal server error |

### Error Response Format

Errors are returned using the RFC 7807 Problem Details format:

```json
{
  "type": "about:blank",
  "title": "Conflict",
  "status": 409,
  "detail": "User already has this role",
  "instance": "/api/auth/user/role"
}
```

For validation errors (`400`):

```json
{
  "type": "about:blank",
  "title": "Validation failed",
  "status": 400,
  "detail": "Email must be a valid email address",
  "instance": "/api/auth/register"
}
```
