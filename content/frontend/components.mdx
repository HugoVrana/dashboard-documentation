# Components

## Dashboard Components

Located in `app/dashboard/components/`.

### Metric Cards

Displays key statistics on the dashboard overview.

```tsx
// app/dashboard/components/dashboard/cards/cards.tsx
import { Cards } from '@/app/dashboard/components/dashboard/cards/cards';
```

Shows four metrics: total revenue, pending invoices, paid invoices, and customer count. Each card fetches its own data from the API.

### Activity Feed

Real-time activity stream powered by WebSocket. Displayed on the home page.

```tsx
// app/dashboard/components/activity/activityFeed.tsx
import { ActivityFeed } from '@/app/dashboard/components/activity/activityFeed';
```

Connects to the WebSocket at `/ws` and displays events as they arrive. Initial events are loaded from `GET /api/v1/activity/recent`.

## Invoice Components

Located in `app/dashboard/invoices/components/`.

### Invoice Table

Paginated invoice list with search and actions.

```tsx
// app/dashboard/invoices/components/table/tableWithPermission.tsx
```

Renders conditionally based on the user's grants — edit and delete buttons are shown only if the user has `dashboard-invoices-update` and `dashboard-invoices-delete` respectively.

### Search

```tsx
// app/dashboard/invoices/components/search.tsx
```

Updates the `query` URL parameter on input, triggering a new search request.

### Pagination

```tsx
// app/dashboard/invoices/components/pagination.tsx
```

Updates the `page` URL parameter. Total pages are derived from the search response.

## Component Patterns

### Server vs Client Components

```tsx
// Server Component (default) — can fetch data directly
export default async function ServerComponent() {
  const data = await fetchData();
  return <div>{data}</div>;
}

// Client Component — required for state, effects, and browser APIs
'use client';
export function ClientComponent() {
  const [state, setState] = useState();
  return <button onClick={() => setState('clicked')}>Click</button>;
}
```

### Permission-Gated Rendering

Components that depend on grants check the user's session server-side and conditionally render action buttons or redirect to `/auth/unauthorized`.
