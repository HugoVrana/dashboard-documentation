# Pages & Routes

## Route Structure

```
app/
├── page.tsx                        # / (home with activity feed)
├── homeContent.tsx
├── auth/
│   ├── login/page.tsx              # /auth/login
│   ├── unauthorized/page.tsx       # /auth/unauthorized
│   └── roles/page.tsx              # /auth/roles
└── dashboard/
    ├── (overview)/
    │   └── page.tsx                # /dashboard
    └── invoices/
        ├── page.tsx                # /dashboard/invoices
        ├── create/page.tsx         # /dashboard/invoices/create
        └── [id]/
            ├── page.tsx            # /dashboard/invoices/:id
            └── edit/page.tsx       # /dashboard/invoices/:id/edit
```

---

## Home

**Route**: `/`

Landing page with a welcome message, a link to the dashboard, and a real-time activity feed powered by WebSocket.

### Features
- Welcome hero section
- Link to `/dashboard`
- Activity feed (live updates via WebSocket at `/ws`)

### API Calls
- `GET /api/v1/activity/recent` — initial activity events (default limit: 50)
- WebSocket `/ws` — subscribes to live activity events

---

## Dashboard Overview

**Route**: `/dashboard`

Main dashboard page displaying key metrics and recent data.

### Features
- Total revenue card
- Pending invoices count and amount
- Paid invoices count and amount
- Customer count
- Revenue chart (monthly)
- Latest invoices list

### API Calls
- `GET /api/v1/invoices/count?status=pending`
- `GET /api/v1/invoices/count?status=paid`
- `GET /api/v1/invoices/amount?status=pending`
- `GET /api/v1/invoices/amount?status=paid`
- `GET /api/v1/customers/count`
- `GET /api/v1/revenues/`
- `GET /api/v1/invoices/latest`

### Layout

```
┌─────────────────────────────────────────────────────┐
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐  │
│  │ Revenue │ │ Pending │ │  Paid   │ │Customers│  │
│  │  Card   │ │  Card   │ │  Card   │ │  Card   │  │
│  └─────────┘ └─────────┘ └─────────┘ └─────────┘  │
│                                                     │
│  ┌─────────────────────┐ ┌───────────────────────┐ │
│  │   Revenue Chart     │ │   Latest Invoices     │ │
│  │     (Monthly)       │ │         List          │ │
│  └─────────────────────┘ └───────────────────────┘ │
└─────────────────────────────────────────────────────┘
```

## Invoices List

**Route**: `/dashboard/invoices`

Paginated invoice management page.

### Features
- Search by customer name or invoice details
- Pagination controls
- Status badges (`pending` / `paid`)
- Edit and delete actions
- Create invoice button

### Query Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `page` | number | Current page (default: 1) |
| `query` | string | Search term |

### API Calls
- `POST /api/v1/invoices/search` — paginated search
- `DELETE /api/v1/invoices/{id}` — delete from table

---

## Invoice Detail

**Route**: `/dashboard/invoices/[id]`

Displays full details for a single invoice.

### Dynamic Segment
- `[id]` — Invoice ObjectId

### API Calls
- `GET /api/v1/invoices/{id}`

---

## Create Invoice

**Route**: `/dashboard/invoices/create`

Form to create a new invoice.

### Form Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| Customer | Select | Yes | Customer dropdown (loaded from API) |
| Amount | Number | Yes | Invoice amount |
| Status | Select | Yes | `pending` or `paid` |

### API Calls
- `GET /api/v1/customers/` — populate customer dropdown
- `POST /api/v1/invoices/` — submit form

---

## Edit Invoice

**Route**: `/dashboard/invoices/[id]/edit`

Pre-filled form to edit an existing invoice.

### Dynamic Segment
- `[id]` — Invoice ObjectId

### API Calls
- `GET /api/v1/invoices/{id}` — load current values
- `GET /api/v1/customers/` — populate customer dropdown
- `PUT /api/v1/invoices/{id}` — submit changes

---

## Login

**Route**: `/auth/login`

Authentication page with email/password and TOTP support.

### Form Fields

| Field | Type | Required |
|-------|------|----------|
| Email | Email | Yes |
| Password | Password | Yes |
| TOTP code | Number | Conditional (if 2FA enabled) |

### Features
- Login and register tabs
- TOTP setup and verification steps
- Error messages
- Redirect to `/dashboard` after login

---

## Roles & Grants

**Route**: `/auth/roles`

Role and grant management for administrators.

### Features
- List of roles with their assigned grants
- Create new grants
- Manage per-role permissions

---

## Unauthorized

**Route**: `/auth/unauthorized`

Access denied page shown when the user lacks a required grant.

### Features
- Error message
- Back to dashboard link
- Login link

---

## Layouts

### Root Layout
Applied to all pages:
- HTML structure with theme and i18n providers
- Font loading
- Dev overlay (development only)

### Dashboard Layout
Applied to `/dashboard/*`:
- Side navigation
- Theme toggle
- Language toggle

---

## Route Protection

Dashboard routes are protected via server-side session checks. Pages redirect to `/auth/login` if there is no valid session, or to `/auth/unauthorized` if the user lacks the required grant for the resource.
