# Data Access Layer

API clients are split by feature area. Dashboard API clients live in `app/dashboard/dataAccess/`, auth service clients in `app/auth/dataAccess/`, and shared clients in `app/shared/dataAccess/`.

Each feature has a server client (used in server components and server actions) and, where needed, a browser client (used in client components).

## Server Actions

Server actions handle form submissions and mutations. They live in `app/auth/actions/` and `app/dashboard/actions.ts`.

### Auth actions (`app/auth/actions/`)

| File | Exported actions |
|------|-----------------|
| `loginActions.ts` | `initiatePkceAction`, `loginAction`, `completePkceCallbackAction`, `completeMfaLoginAction` |
| `totpActions.ts` | TOTP setup and verification |
| `userActions.ts` | Block, unblock, delete, reset password, resend verification |
| `roleActions.ts` | Create, update, delete roles; add/remove grants |
| `grantActions.ts` | Create, update, delete grants |

### Login flow

The login flow uses server actions rather than a direct form POST:

1. `initiatePkceAction(url)` — generates a PKCE code verifier, stores the state in an encrypted httpOnly cookie, returns the `request_id` to the login form.
2. `loginAction(email, password, url, requestId)` — submits credentials to the OAuth2 server. Returns `{ status: 'success', ... }`, `{ status: 'mfa_required' }`, or `{ status: 'error', message }`.
3. If MFA is required, the MFA token is stored in an encrypted httpOnly cookie and the user is shown the TOTP input. `completeMfaLoginAction(totpCode)` completes the flow.
4. On success, the access and refresh tokens are stored in the NextAuth session.

### Dashboard actions (`app/dashboard/actions.ts`)

Handles invoice create, update, and delete form submissions.

## Permission Checking

Grants are flattened into the NextAuth JWT at login time. Permission checks read from the session rather than making API calls:

```typescript
import { auth } from '@/auth';

export default async function ProtectedPage() {
  const session = await auth();
  const canCreate = session?.user.grants.includes('dashboard-invoices-create');

  return (
    <div>
      {canCreate && <CreateButton />}
      <InvoiceList />
    </div>
  );
}
```
