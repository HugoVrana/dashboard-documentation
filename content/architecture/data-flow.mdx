### Step-by-Step

1. **User Login**: Client sends credentials to OAuth service
2. **Token Generation**: OAuth validates credentials, generates JWT + refresh token
3. **Token Storage**: Frontend stores tokens (NextAuth session)
4. **API Requests**: Frontend includes JWT in Authorization header
5. **Token Validation**: Backend validates JWT, extracts grants
6. **Access Control**: `@PreAuthorize` checks user permissions

### Request Processing

1. **Filter Chain**: `JwtGrantsFilter` extracts and validates JWT
2. **Security Context**: User grants loaded into Spring Security context
3. **Authorization**: `@PreAuthorize` annotation checks required grants
4. **Controller**: Handles request, validates input
5. **Service**: Business logic execution
6. **Repository**: Database operations via Spring Data
7. **Response**: DTO mapping and JSON serialization

### Step-by-Step

1. **Connect**: Client opens WebSocket via SockJS endpoint (`/ws`)
2. **STOMP Handshake**: Client sends STOMP CONNECT frame
3. **Subscribe**: Client subscribes to topics (`/topic/metrics`, `/topic/invoices`)
4. **Receive Updates**: Server pushes messages to subscribed clients

### Topics

| Topic | Update Frequency | Content |
|-------|-----------------|---------|
| `/topic/metrics` | Every 5 seconds | Live dashboard statistics |
| `/topic/invoices` | On change | Invoice CRUD events |
| `/topic/revenue` | On change | Revenue updates |
| `/topic/customers` | On change | Customer activity |

See [WebSocket Messages](/websockets/messages) for detailed message formats.
